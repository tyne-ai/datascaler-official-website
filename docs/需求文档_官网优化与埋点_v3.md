# 第二部分 · 对应数据埋点（方案 A：GTM）

## 2.1 架构 ✅
- 已接 GA4 `gtag.js`（`G-M5NX3Y8KY6`）+ Clarity（`x04rt4ltvq`），仅生产环境启用。
- **加装 GTM 容器**（`layout.tsx` 已埋占位 `GTM-XXXXXXX`，待换真实 ID）。
- 分工原则：**gtag.js 负责 GA4 基础 pageview；GTM 只配 GA4「事件」代码**（GTM 内不要再加 GA4「配置」代码，否则 pageview 重复计数）。
- 已预埋 `track(event, params)`（dataLayer.push）+ `data-cta`/`data-plan` 属性，GTM 上线即接。

## 2.2 跨域追踪（P0，最关键）⬜
`datascaler.ai` → `app.datascaler.ai` 不配跨域，注册来源全变 `(direct)`，归因失效。
- GTM 的 GA4 事件代码配 Cross-domain（两个域）+ GA4 后台 Data Stream 同步。
- ⚠️ 前提：**app 端需装同一 GA4 属性**，否则 linker 无效——与 app 团队确认。

## 2.3 事件清单（只做 P0 + P0.5）

### P0 转化核心（GA4 标记为 Conversion）
| 事件 | 触发 | 参数 | 状态 |
|------|------|------|------|
| `sign_up_click` | 点击**通用注册按钮**「开始体验 / Get started」（Hero / 导航 / FAQ / 底部） | `button_location`(hero/nav/faq/footer) | Hero ✅；nav/faq/footer ⬜ |
| `plan_click` | 点击**定价套餐按钮**（Free/Pro/Team/Enterprise） | `plan_name`、`cta_type`(signup=Free·Pro / demo=Team·Enterprise) | ✅（含 `data-plan`） |
| `demo_request_submit` | `/contact` 表单提交成功 | `form_type=demo`、`locale` | ⬜ |

> 主转化标记：`sign_up_click` + `plan_click(cta_type=signup)` + `demo_request_submit`。

### P0.5 高价值意向
| 事件 | 触发 | 参数 | 状态 |
|------|------|------|------|
| `demo_cta_click` | Hero 次级「预约演示 / Book a demo」（非定价区的 demo 入口） | `button_location=hero` | ✅ |
| `hero_brand_search` | Hero 品牌域名输入框输入并提交 | `has_query`(bool，不传域名避免 PII) | ⬜ |

> 本期不做 P1/P2（scroll_depth、social_proof_view、pricing_toggle、faq_expand、blog_read、lang_switch、outbound_click、nav_click 等）；定性行为看 Clarity。

## 2.4 需研发补的 dataLayer 调用 ⬜
